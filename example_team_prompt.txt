Create an agent team with 3 teammates to fix testing failures in parallel:

  Teammate 1 — Frontend Fixer:
    Own files: src/components/
    Settings: isolation: worktree
    Tasks:
    1. Run tests across the components folder
    2. Identify and fix failing tests

  Teammate 2 — Backend Fixer:
    Own files: src/api/
    Settings: isolation: worktree
    Tasks:
    1. Run tests across the API folder
    2. Identify and fix failing tests

  Teammate 3 — E2E Tester:
    Settings: isolation: worktree
    Tasks:
    1. Wait for Teammates 1 and 2 to complete their tasks
    2. Run the full End-to-End test suite
    3. Log findings

Rules:
- Give agents the correct path to the workspace.
- Make sure to specify `isolation: worktree` so each agent gets their own git branch and avoids file conflicts!
- Wait for all teammates to finish, then gather their results.
- CRITICAL: When the tasks are complete, execute a cleanup command `git worktree remove` to delete the temporary teammates' worktree directories and branches!
- CRITICAL TEAM LIFECYCLE: You MUST actively monitor your inbox. If a teammate sends you a permission request to use a tool, you MUST immediately approve it using the `sendMessage` tool so they do not hang indefinitely!
- Use `TaskCreate` and `taskUpdate` to enforce strict state machines (pending -> ready -> claimed -> in_progress -> complete). Do not mark tasks as ready until their blockers are explicitly resolved.
- Use `shutdown_request` and `shutdown_response` to cleanly terminate the team when finished.
